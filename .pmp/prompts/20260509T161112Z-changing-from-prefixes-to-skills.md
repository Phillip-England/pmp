# Changing from prefixes to skills

Right now a prefix is just some text that is prefix to your compilation phase, but we might need different prefixes each types so like we might create one prefix for the way that the UI is managed and we might create another prefix to explain other parts about the project prefixes are no longer called prefixes they called skills and skills are essentially just bodies of text that you want included with every prompt in the project that's essentially what a skill is and whenever you compile your project, those skills are kind of included in the prompt in the same way that the prefixes now, but we want to migrate over from this idea of a prefix to a skill instead.
